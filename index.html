ETS: ESP32 Tracking system

The purpose of this project is to sniff probe request packets sent by smartphones that are looking for a Wi-Fi connection. From each sniffed packet some information will be extracted and elaborated in order to do several analysis.

The firmware of the ESP is written with the purpose to sniff probe request packets sent by smartphones that are looking for a Wi-Fi connection. From each sniffed packet some information will be extracted. After each minute these information are sent to a server that process and store them in a MySQL database. Then is possible to see the processed information, like real time location of the smartphones, smartphone frequency and etc., through a GUI.

The system is also able to recognize random generated MAC address: in order to improve security some devices uses random generated MAC address to send probe request packets. Briefly a random generated MAC address can be recognized because the 2nd bit of the 1st byte is set to one. A class is able to exploit a leak in this mechanism and group all tuples with different MAC addresses belonging to the same device.

Source code available on my GitHub. A demonstration video on youtube shows how ESP32 works. Check out also this short presentation.

Study and design data models for NFV and SDN architectures

Research collaborator at Politecnico di Torino in collaboration with PoliTO Netgroup (Computer Networks Group at Politecnico di Torino) in order to study and design data models for NFV and SDN architectures. Worked side by side with two PhD students and the professor of "Distributed Programming II" course at PoliTO.

Proposed a new data format (described by means of an XML schema) for the representation of all the most relevant information in the NFV (Network Function Virtualization) and SDN (Software Defined Networking). Also a RESTful web service has been implemented in order to store and retrieve the NFV/SDN information.

PoliTO Netgroup official GitHub repo
